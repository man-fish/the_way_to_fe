# git 分支 

> 刚学到这里的时候概念是有点模糊的，原因大概是没有经历过实际开发过程，总之消化了好长一段时间。

#### 分支的概念

> 因为版本控制对我来说不太实用，就拿我现在的阶段的分离开发来举例，做一个完整的**web**项目，首先要为项目申请一个**git**仓库，之后我提交**readme**文件（**没有一次提交是不能创建分支的**）此时会自动创建主分支(**MASTER**)，这之后我创建client的分支来开发前端，在创建**server**的分支来开发服务器，不同的分支之间不会相互影响，最后前后端构建完毕之后我们可以合并到主分支当中。	

这个过程用实际的代码可以说是更加的易于理解。

```bash
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning
$ git init
#创建项目
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ touch readme

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git add readme

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git commit -m '项目开始了'
[master (root-commit) 60eb338] 项目开始了
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 readme
#开始项目并且创建readme文件，执行第一次提交。
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git branch client
#创建分支client（前端）
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git branch server
#创建分支server（服务器）
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git checkout client
Switched to branch 'client'
#进入分支前端
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (client)
$ mkdir client

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (client)
$ touch ./client/package.json

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (client)
$ git add ./client/package.json

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (client)
$ git commit -m '前端构建完毕'
[client ee81db9] 前端构建完毕
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 client/package.json
#假设前端构建完毕
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (client)
$ git checkout server
Switched to branch 'server'
#进入分支后端
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (server)
$ mkdir server

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (server)
$ touch ./server/app.js

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (server)
$ git add ./server/app.js

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (server)
$ git commit -m '服务器构建完毕'
[server eb10bc6] 服务器构建完毕
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 server/app.js
#假设后端创建完毕
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (server)
$ git ls-files
readme
server/app.js
#此时后端分支仓库中的文件

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (server)
$ git checkout client
Switched to branch 'client'

Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (client)
$ git ls-files
client/package.json
readme
#此时前端分支中的文件
##我们可以看出前后端的仓库是分离的。这种同一结点的双向分支可以说是分支功能的应用之一。
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (client)
$ git checkout master
Switched to branch 'master'
#切换到主分支
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git ls-files
readme
#主分支还没有进行任何操作
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git branch
  client
* master
  server
#查看所有分支
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git merge client server
Fast-forwarding to: client
Trying simple merge with server
Merge made by the 'octopus' strategy.
 client/package.json | 0
 server/app.js       | 0
 2 files changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 client/package.json
 create mode 100644 server/app.js
#合并两个分支到主分支
Administrator@DESKTOP-S39Q6LU MINGW64 /F/git_leraning (master)
$ git ls-files
client/package.json
readme
server/app.js
#主分支已经有两个分支的所有修改更新。
```
