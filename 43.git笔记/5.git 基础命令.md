# git 命令合集

### Ⅰ.新建git仓库

##### 1.把已有项目代码纳入Git管理

```shell
$ cd 项目文件夹
$ git init 

#这将创建一个名为的新子目录.git，其中包含所有必需的存储库文件-Git存储库框架。至此，您的项目中尚未跟踪任#何内容。（有关刚创建的目录中确切包含哪些文件的更多信息，请参见Git Internals.git。）
#如果要开始控制现有文件的版本（而不是空目录），则可能应该开始跟踪这些文件并进行初始提交。您可以使用一些#git add命令来指定要跟踪的文件，然后加上一个命令来完成此操作git commit：

$ git add *.c
$ git add LICENSE
$ git commit -m 'initial project version'
```

##### 2.把要新建的项目代码直接用Git管理

```bash
$ cd 某个文件夹
$ git init git_learning
Initialized empty Git repository in F:/SSR/git_learning/.git/
#这里会创建一个和你项目名相同的文件夹

```

##### 3.下载一个项目

```shell
# 下载一个项目和它的整个代码历史
$ git clone [url]
```



### Ⅱ.项目的配置

Git的设置文件为`.gitconfig`，它可以在用户主目录下（全局配置），也可以在项目目录下（项目配置）。

##### 1.显示Git配置 git config

```bash

$ git config --list

$ git config --local --list
core.repositoryformatversion=0
core.filemode=false
core.bare=false
core.logallrefupdates=true
core.symlinks=false
core.ignorecase=true
user.name=innotest
user.email=893930581@qq.com

#显示主目录下的用户全局配置
$ git config --global --list
user.name=inno
user.email=893930581@qq.com
```

##### 2.更改Git配置 git config

```bash
# 编辑Git配置文件
$ git config -e --global

# 设置提交代码时的用户信息
$ git config --global user.name "[name]"
$ git config --global user.email "[email address]"
```

### Ⅲ.暂存区的操作

> 文件在工作区和暂存区之间的操作。

##### 1.添加文件到暂存区 git add

```
# 添加指定文件到暂存区
$ git add [file1] [file2] ...

# 添加指定目录到暂存区，包括子目录
$ git add [dir]

# 添加当前目录的所有文件到暂存区
$ git add .

# 添加每个变化前，都会要求确认
# 对于同一个文件的多处变化，可以实现分次提交
$ git add -p
```

##### 2.删除暂存区中的文件 git rm

```cmd
# 删除工作区文件，并且将这次删除放入暂存区
$ git rm [file1] [file2] ...

# 停止追踪指定文件，但该文件会保留在工作区
$ git rm --cached [file]

# 改名文件，并且将这个改名放入暂存区
$ git mv [file-original] [file-renamed]
```

### Ⅳ.仓库的操作

> git commit只能将暂存区的数据提交到仓库

##### 1.提交 git commit

```shell
# 提交暂存区到仓库区（默认是全部提交）
$ git commit -m [message]

Administrator@DESKTOP-S39Q6LU MINGW64 /f/SSR/git_learning (master)
$ git commit -m 'Add readme'
[master (root-commit) 5936b15] Add readme
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 readme


# 提交暂存区的指定文件到仓库区
$ git commit [file1] [file2] ... -m [message]

# 提交工作区自上次commit之后的变化，直接到仓库区
$ git commit -a

# 提交时显示所有diff信息
$ git commit -v

# 使用一次新的commit，替代上一次提交
# 如果代码没有任何新变化，则用来改写上一次commit的提交信息
$ git commit --amend -m [message]

# 重做上一次commit，并包括指定文件的新变化
$ git commit --amend [file1] [file2] ...
```



### Ⅴ.查看状态
